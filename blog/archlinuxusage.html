arch-linux-usage

[link—standalone] Whoops. I did a thing. I decided to give Arch a go.


I've never seriously used Arch Linux on my own hardware. I've played around with a VM, run Antergos before (hilariously it was my primary machine while I was job searching, even though it was pretty ridiculous), but never seriously used Arch long-term. I installed it on my test laptop a week ago, and with no outstanding issues, dropped it on my primary machine. Woah.


It's pretty neat. There's not much to say - it's not an opinionated distro. That's my main problem with Ubuntu and others - they've got their logo plastered all over the OS and it's got their colorscheme everywhere. Debian and Fedora tend to be my favorites, but Fedora's "I'm not a rolling release seriously trust me plz" model is really weird to me. Plus dnf is sloooooooow.


Arch is mostly what I want. I'll probably dabble with Parabola because ~FREEDOM~, but I'm really enjoying a rolling release with both current packages and no opinionated branding. Arch is basically 100% upstream with no tweaks and no changes. It doesn't feel like a cohesive unit, but more like a minimal system with the option of adding a bunch of extra packages on top. The installation just requires you to partition things the way you want, and pretty much does the rest for you, with a ton of really useful tools along the way. Pacstrap is like a good version of a Gentoo stage3, wifi-menu is like an easy version of wpa_supplicant, and pacman is...
well....


Pacman is the one area where arch falls down. It's fast and configurable and really easy to work with, but man is it a little sticky. When using dnf to remove a package, it removes its unneeded dependency packages at the same time. Debian's apt will mark them as such, and you can 'autormeove' them later. Here's three commands that all do the same thing:


1) apt autoremove
2) dnf autoremove
3) pacman -R $(pacman -Qtdq)


Seriously, I don't think it'd be that hard to write a package manager with syntax that uses actual words instead of switches. That is my one major complaint about Arch. But hey, at least it's fast, mirrors are easy to switch between, and its keyring system is very simple to understand!

------


Anyway, Arch is a great distro. No SELinux or AppArmor by default, but that's fine. Nothing but the barebones, raw, lean, simple setup that you need, and then you can add what you want after. I'm a tentative convert. I know it's not a useful system to be using, but I've cut my teeth on Debian- and Fedora-based systems for a while now, and I think it's fine to use this as long as it doesn't get in my way. The minute I have to struggle with something simple like printing or nework config, it's going in the trash and I'm going back to Fedora or Debian. But using a full DE like Gnome3 (it's really really fast now!) mitigates any issue I have. I encourage any readers to try it, becuase it lets you do what you want, instead of dumping a full set of gnome-games on you (debian) or preinstalling an office suite and mail/chat/calendar programs you won't use.


I'll add a script later to get things set up, but getting a new Arch system running to satisfaction is really easy. Just a few pacman, useradd, and systmectl commands, and you've got a full usable system, with wifi, printing, bluetooth, power management, and all that.
