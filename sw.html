Software I Use/Recommend

Software I like, all in one concise list. (My version of suckless.org's "rocks" page)

  • GNU/Linux Operating Systems: clearly the only reasonable choice. There are many varieties (distributions, or "distros") but if you don't know better stick to Ubuntu.
    • Debian is lighter and faster than anything else I've used while still being easy to setup as a Desktop and Server - anyone who says Debian's not a good Desktop OS hasn't tried it. Truly legendary.
    • Fedora is the sensible choice for up-to-date software and mainline kernels.
    • Arch Linux is as light as Debian but current and sometimes breaks if you don't run a super lean system. It's a niche use case but respectable.
    • Ubuntu is Debian but easier, more bloated, and with a more consistent release schedule. Since it's widely used it's great for new users, but more experienced users should use Debian.
    • Anything else that's not a boutique distro is probably fine. There isn't a use case one of the above won't cover.
  • Decentralized Services
    • email is a good way to communicate if you don't care about metadata security, which you should.
    • Matrix is a great current "next gen" messenging service - encryption, decentralization, open standards, current development, etc. The only problem is it's hard to run a server.
    • Cryptocurrencies like Monero and Bitcoin are an alternative to fiat currency that don't depend on your government being truthful. We should use these as a replacement for USD if possible.
    • Tor is a decentralized obfuscation network, without much drawback. Use Tor Browser whenever possible, within reason, to support those who need it, even if you don't.
  • Window Managers: In 2016 I started using Window Managers instead of full-on Desktop Environments. Most are highly configurable, lightweight, and modular. I recommend Sway, i3, and IceWM.
  • Portable Shell Scripting: most "code" I write is in bash (or POSIX sh if I can manage). Although not as elegant as something like Powershell and not as fast as C, piping text streams is a beautiful design and shell scripts work on every operating system.
  • BSD Operating Systems: a more specialized choice for the advanced user. FreeBSD is used heavily by Netflix and Sony. OpenBSD is my desert-island operating system and possible the most interesting computing project today. (It runs this website.)
  • Firefox: Not great out of the box, but it's the only decent option. Other options contribute to the browser monoculture or are spyware. The next best option is Lynx. Brave is great out of the box, but is Chromium. I've been recommending this to less technical people and it works well for them.
  • Python: a fun language to learn. Good defaults, fast enough, portable.
  • Vim is an extensible and standard text editor that will be your favorite program if you use text files a lot. My website is text files. My notes are text files. Code is text files. Configuration and logs are text files. Vim all the things!
  • Tmux is a window manager for the shell, or screen++. It can be used with vim or with practically anything, and for any reason. I put it everywhere because you will always find a reason to use it. Honorable mention: byobu, which is the Ubuntu-fication of screen and tmux.
  • Photopea I've not used for a number of years (GIMP is good enough for anyone) but it's a fully functional photoshop clone in the browser for free.
  • Podman is an excellent container management tool that is compatible with Docker, runs daemonless, and is the way that RHEL8 and Fedora manage containers. There's also a great interface for it in...
  • Cockpit! Web administration tech has been around for a while but cockpit is just the best. Runs on darn near every Linux OS, frequently updated, and can do basically everything you'll need to do on a server - networking, firewall, software configuration, SELinux, VMs, and even a built in terminal. Solid, reliable, and always getting better.
  • Coreboot The best BIOS around. I run it on everything I can. Some System76 machines and PC Engines boards run it by default. Its fully-free, blobless downstream fork LibreBoot used to be a good option but I can't recommend it anymore as it seems to be a dead project.

Software I Recommend Not Using

I have either used this extensively or very little, and recommend avoiding it. They usually have cascading and far-reaching negative effects.

  • Appliance and "vendor-purpose" operating systems. They trick you into doing the wrong thing.
  • Any "Note-Taking" software. You're better off storing stuff in a format you can bring with you anywhere.
  • Programming language-specific package managers, like NPM or Pip or Go's get. They encourage dependence and weakness. There is rarely any appropriate occasion for their use. They make using the software nearly impossible, especially if they don't have a strong Internet connection, or dependencies change.
  • Software that encourages vendor or OS lock-in, especially to non-free operating systems. Example: iMessage and Facetime are great examples of technologies that are only available if you buy a specific piece of appliance hardware and run a specific non-free OS. I've also in the past used my NAS's built-in software, but since it was not transparent and portable to another brand of NAS (or a standard operating system) I switched to more standardized software in containers instead.
  • Easy Buttons. Good software should be simple and transparently expose the power of its subsystem to you, while allowing you to gloss of the the more complex aspects while you're still a beginner.
