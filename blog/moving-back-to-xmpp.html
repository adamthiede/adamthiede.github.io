History Time

Over the past several years I've somewhat aggressively pushed everyone in my life to use Signal, since it's generally the easiest to use messenger program that checks most of my boxes. It is encrypted, open source, cross-platform, and has clients for desktop computers. Despite my belief that the cracks are beginning to show, Signal remains a safe, secure messenger that is available to everyone with an Apple iPhone (with associated iCloud account) or a Google Android Product (with associated Google account).

However, it doesn't fit the more important critera that I've been ignoring. The server is centralized, so I can't run my own if I so choose. The server software is only marginally open source (for what I belive is a stupid reason). The Signal team is openly hostile to third-party clients, will not support decentralization, the service requires a cell phone number for sign-up, and the desktop clients are dependent on the smartphone client. These are non-concerns for most people right now, but they always end up being the killer anti-features.

When a platform is centralized and does not use open standards, it will go away, and it will hurt when it does. Remember AIM? ICQ? Google Talk? When these systems shut down, they forced you to move to a new platform, regardless of if you were ready or not. When Signal inevitably does something else to close their system down, bakes in some boneheaded cryptocurrency scheme, or decides to remove support for de-Googled Android custom ROMs, what then? Can we use an old client, move our server, all without effecting our contacts? No dice. When Signal says jump, the users' only option is "how high".

If you wonder why I fret about chat client so much, it's because I want the primary way I communicate with friends and family to be safe, secure, and as personal as if we were face to face. I don't want anyone but me and the recipient to see the message. That's it. That's the one thing that's important. You have no idea how difficult it is to ensure that one requirement.

Personal Complaints

Besides the problems with the larger organization and system as a whole, I have personal issues with Signal.

  • They do not ship a decent desktop client. It's made of web junk and painful to run on my low-resource computers.
  • Despite being made of web technologies, they do not ship a "web" client like WhatsApp or Telegram, which would mitigate the following issue:
  • They only ship x86_64 builds of their desktop client, so users of the (now >year-old) M1 Macbooks, or any Windows or Linux ARM devices cannot use the desktop client. This also means no Pinephone client.
  • They do not officially support non-Debian Linux. Flatpaks and Snaps are unofficial builds by Internet randos. Building it from source is painful.
  • It's not possible to register without a phone number.
  • It's not possible to set up Signal on a new computer without "approving" it with the phone first. The desktop client can be used with the phone completely off, though.
  • Installing Signal without identifying yourself to Apple or Google is possible but highly discouraged. The Android standalone download is literally labeled "Danger Zone". They also refuse to distribute Signal via F-Droid, the good Android app store.
In short, it's heavily smartphone-dependent, which means it's also heavily Apple and Google dependent. How they ship an encrypted messenger and laud the benefits of privacy - while simultaneously ensuring their customers also have to submit to surveillance by these 2 tech giants in order to user their product - baffles me.

Where Do We Go?

There's a lot pushing me away from Signal. The only thing keeping me there is the same thing that keeps people on Facebook - the network effect. If Signal were decentralized like email, I could choose my own clients, run my own server, and solve the problem for myself, without effecting others. This is why I must flee to a decentralized messenger on an open standard - the next time I have complaints about what my messaging client does, I can swap it out and my recipients aren't inconvenienced. This is why I'm not telling everyone to switch to Telegram or WhatsApp - they have all the same issues Signal does, and more.

There are a bunch of options for decentralized, self-hosted messengers out there, but the one with staying power is XMPP (aka "jabber"). It has served as the backbone for a bunch of proprietary messengers, like Cisco Jabber and Google Talk, so it's battle-tested. There are many different clients and servers for every platform under the sun, and many are lightweight and resource friendly. The reason I don't want to migrate to Matrix is that it's too new, the one (!) server implementation is a resource hog, and out of all the clients, the only generally competent one is written in Electron, for a matching waste of energy on your server and client.

Using decentralized systems, or systems under your own control, mitigates the eventuality of proprietary or centralized systems turning sour. I'm running my own XMPP server on this webserver right now, using prosody. It is almost trivially easy to set up, can run on minimal system resources, and for extra security, SSL certs are free via LetsEncrypt. For those unwilling to go to the command line, YunoHost bundles it in the default setup.

Signal has been great. I've enjoyed using the program from its early days, but using what amounts to a proprietary system has restricted my freedoms long enough. I want to be able to run my own client. I want to be able to run my own server. I want to control my data. Signal affords me none of these freedoms. XMPP does. Some people see the availability of these choices as "fragmentation", but I see it as a good thing. What I want out of a chat program may not be what you want, but at the very least, our different clients should speak the same language.

I hope to see you there.
