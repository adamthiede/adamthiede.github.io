User-Friendly Information Security Advice

Hello there! You have a computer or two, and you want reasonable, understandable advice to make it and your activity on it more secure? This in-depth guide is a good starting point.

The basics of Information Security:

  • Confidentiality - information is only accessible by those authorized.
  • Integrity - information is accurate, complete, and not changed by the computer system on which it resides.
  • Availability - information is available easily when needed and expected.

Fundamentally, security is about control. You need to have as much control over your computing environment as you possibly can - networking, software, hardware, even physical access. The more variables you add in, the less likely you are to have control over your data. So the goal is to keep things simple, but not quick. This cannot be done in 20 minutes. If I link to something, it's usually to explain jargon, so I highly recommend you read it until you have a decent understanding.

This guide assumes you're using Windows and unable to use anything else. It is written largely with the needs of a specific person in mind, so it may not apply to you.

Compartmentalization

Separate Computer

Ideally, you have two separate computers that are never on your network at the same time - one is your personal computer that runs your secure suite of trustworthy programs and is as locked down as you can make it. It has personal data. The other is completely untrusted, you don't log into any personal accounts, and you don't care what happens to the data on it. A good way to maintain this compartmentalization is a to use a guest wi-fi network on your untrusted device. If you cannot control your own network, also try shutting down the personal system before turning on the untrusted system, or unplugging your personal desktop's ethernet cable while your untrusted device is on the wi-fi. The untrustworthy system should not know of the existence of the trustworthy one, and there should be no chance of them "talking" on the network.

Your secondary system can be an inexpensive used laptop. Many are available for $200 or less somewhere like Craigslist or eBay. I recommend looking for a Thinkpad X250/T450 or a recent Dell Latitude. They are reasonably performant, get BIOS updates as needed, are built well, and they're relatively inexpensive on the used market.

Virtual Computer

If you don't have or cannot obtain a second computer, you can have another computer as an application inside your computer via a program called VirtualBox. VirtualBox is a program that can run multiple full operating systems on Virtual Machines like a program on your computer. Go through the offline install of Windows again, but this time in VirtualBox. This virtual computer will be your "sandbox".

Important note - you should not make your trustworthy system a virtual machine that resides on the untrustworthy system. The host system can view and manipulate the contents of the virtual machine. The host system should be trustworthy.

Virtual Machines use Virtual Hard Drives, and with that comes the concept of "snapshots". You can save the state of the virtual machine at any point in time, continue working, and if you decide to revert your changes, you can from any point in the future. When you complete the install and do all the updates, create a snapshot. It will be the "base" state. From here, any time you need to do something like open a questionable file, visit a sketchy website, or run a program that isn't trustworthy, you do it from within the virtual machine, and revert to the snapshot when you finish, to completely erase the evidence, or ill effects of any damage. If it's a program that requires access to a hardware device (say, iTunes for syncing an iPod) you can "pass through" the USB devices to the virtual machine.

The super-advanced way to do real compartmentalization is Qubes. It's what Edward Snowden uses. For most people, it's overkill on security and not usable enough.

Software

Keeping a system secure means keeping bad or mysterious software off of it. Start with a clean, offline installation of your operating system; Windows especially - it tries to trick you into using a Microsoft account, and that is not what you want. Before connecting the system to the Internet, remove all the pre-bundled software, and let the system sit for a while with task manager open, until usage settles down. Windows' first boot runs a bunch of background tasks.

If you're security-conscious, there are very few programs you should install over the base system. The basics you install should be either replacing base programs with something lighter-weight and open-source (Firefox and SumatraPDF), or adding necessary functionality (7-zip and VLC). A good set of basics for Windows can be installed via Ninite, an automated batched program installer. Don't install Chrome - that is explained later.

On macOS, Firefox and VLC should be all you need.

The untrustworthy system should be the home for mystery-meat software - programs that are not Open Source. If your primary use case for your computer requires more powerful hardware (professional creative work, engineering, industrial applications, playing video games, running network services) then consider making the secure system a cheaper used laptop. The programs used in these industries - Autodesk, Adobe, and pretty much all video games - typically require invasive access and cannot be considered trustworthy. Play video games on consoles that are not connected to your network.

Web Browsing

Modern computing is done mostly through the browser. It is necessary to keep your browser secure and up to date. Sadly, the safest browsers to use are the mainstream ones, so pick between Firefox, Chromium, or Brave. Do not use smaller hobby projects like Waterfox or Pale Moon as they do not get timely security updates. Do not use Chrome. It cannot be treated as secure as it is not Open Source. Chromium, the open source project on which Chrome is based, is difficult to obtain and usually doesn't update itself on Windows or Mac, so don't use it unless you keep up with releases and update it yourself.

Browsing the web in the modern world is not an activity that is secure by default. The reason most technical users block ads is not because they are opposed to seeing the ads - ads these days include embedded programs that track your browsing and behavior around the Internet, and can compromise security even on reputable pages, like big mainstream news sites.

Good browser add-ons, in order of importance, are below.

  • uBlock Origin, and not "uBlock" (a knockoff that does not keep you safe). In uBlock Origin's settings, enable as many of the filter lists as you feel comfortable. The default set is fine, but more is better.
  • HTTPS Everywhere encrypts all eligible sites and warns you when visiting an unencrypted non-HTTPS site. (Keep in mind HTTPS does not mean "trustworthy", it just means the page you see is verifiably the same page that the website put up, and there is no "man in the middle". Bad actors can have HTTPS too.)
  • Clean URLs strips tracking content appended to URLs and generally makes links more shareable.
  • Privacy Possum attempts to break commercial tracking and ad requests. Generally a good thing to have but not necessary.
Generally running with these on all the time will not break any websites. If sites do break, or ask you to turn off your adblocker, disabling JavaScript on those sites with uBlock Origin will fix that issue. In Firefox, make sure the uBlock Origin add-on is enabled when in Private Mode as well. More advanced users can try out the venerable NoScript which disables JavaScript (programs) from running in your browser. While the above add-ons will not break the functionality of websites, NoScript will. You've been warned.

Make liberal use of Private Browsing/Incognito Mode - it is useful for keeping sessions separate. Clear cookies and browsing data regularly. History is useful to have around, but cache, cookies, and active logins can be cleared frequently.

Passwords

Use an open-source password manager for storing passwords, like KeePassXC, a fully offline file-based password manager, or Bitwarden, a hosted, Internet-accessible password manager. This is non-negotiable, as proprietary solutions, especially cloud-based solutions like LastPass or 1Password, cannot be verifiably secure or private. To maintain security, your passwords must be accessible only to those trusted with the information they unlock.

Lock the password database with a strong passphrase. The only passwords you should remember are the three separate passphrases to decrypt your operating system's boot disk, log into your user account, and unlock your password manager. They should all be different.

Use different randomly generated passwords, or diceware passphrases, for every single online account you wish to keep secure. KeePassXC and Bitwarden can generate these for you.

Use two-factor authentication (2FA) for every account. Text-message based 2FA is not secure and should not be used. KeePassXC can also generate 2FA codes. Smartphone apps can be used to generate 2FA code as as well, but these are less trustworthy systems.

Communication

All the security in the world doesn't mean much if you send sensitive information though unencrypted "cleartext" or backdoored channels. Your priority with messaging should be ensuring integrity and confidentiality - the messages you send should only be visible to the recipient, and they should contain exatly what you wrote in them. The best way to do this is either via a secure, centralized core system, or using federated, decentralized communication protocols.

Decentralized services are usually based on an open, standardized protocol - email is a good example. There exist many email servers and email clients, you can sign up for any server or run your own, and use any client with it. Any user using any email client on any email server can communicate with any other user using a different client on a different server. For example, your friend with a Samsung phone can send an email from his Gmail account via the Samsung email app to your Yahoo account that you check with your iPhone's Mail app. Phone calls are also decentralized - a Verizon iPhone can call a T-Mobile Samsung phone just fine.

Email

Email is common, but you can't make it truly secure. Email can be read by the provider (Google, Microsoft, etc). At some point you will use email, so make it at least a litte more secure: use an email client like Thunderbird to store your emails offline. Email clients can be configured to both display and send plaintext email (no HTML formatting) and not download contents of emails from remote servers, unlike webmail. HTML-format email can download remote images or other files to trick you and can potentially run code on your computer; it's an attack vector you should close off. Email can be encrypted with PGP, but the complexity and drawbacks make it not worth it.

Phones

Chat clients, like WhatsApp, Telegram, and Signal, all require a smartphone, and should be used with hesitation, only in tandem with an appropriately secured smartphone. Any client requiring a phone number can potentially be hijacked by a bad actor - phone numbers are easy to steal. Things like Telegram, WhatsApp, and other messengers should not be treated as secure. The service providers can view and alter the contents of the messages.

Phone calls and regular SMS text messages are not secure. Anyone with technical know-how and about $500 worth of equipment (not only governments and police) can intercept phone calls and text messages.

Matrix and Signal

Signal is a drop-in replacement for messaging and phone calls. It can do video calls, too. Everything is encrypted and secure, and it's incredibly easy to set up and use. The desktop application can do voice/video calls now too, but it must be linked to the smartphone in order to do so. Once that is done, the smartphone can be turned off and you can still use Signal on the desktop, unlike something like WhatsApp Web.

A better option than Signal is Matrix. It does not require a phone to use, which is why I recommend it over Signal. It is a decentralized system like email, but it's more like chat than serialized messages. The best client for Matrix is Element. A simpler client for smartphones is fluffychat. You can pick any server or host your own, but the default is just using the standard Matrix.org home server. The trick is getting those you communicate with to sign up as well, and since Signal is easier, has less opportunity for mistakes, and doesn't make normal people think about encryption keys, it's probably a better option for most.

Smartphones

Whether Android or iOS, fundamentally, these devices cannot be controlled. They are automatically updated by the carriers, notable bad actors, and the user is given so little access to the system such that appropriate lockdown cannot be done. While iPhones sit in a much better position than Android devices, both are systems that should not be considered trustworthy.

However, people like to talk on the phone. People like to use messaging apps. In order to do this securely, you'll likely have to change your behavior regarding your smartphone. Here are a few general tips.

  • Only buy smartphones from notable good manufacturers that have a history of supporting devices with prompt security patches for a long time. This essentially means only buy Apple iPhones or Google Pixel devices. Samsung's Galaxy S and Galaxy Note lines are runners-up. Buying last year's model refurbished is a good option to save a buck.
  • Download as few third-party apps as possible. Whatever you do download should be verifiably open source. Smartphone apps (especially on Android) can escape their sandbox and steal your data, or trick you into doing things you should not. Most people should only have one or two apps aside from what is pre-installed. At any given time, your smartphone's data should be an acceptable loss.
  • Use the phone as little as possible. Smartphones can usually share their data connection via USB or Wi-Fi to a real computer. You are better off bringing a secured laptop with you and tethering it from your smartphone for mobile computing.
  • Only plug the phone into known-good cables and chargers. Harmless-looking cables can snoop on your device. Furthermore, only plug your phone into your charger - don't use a computer's USB port to charge.
  • Messaging apps that require a cell phone number should be carefully vetted. If you are being targeted, it is trivially easy for an adversary to steal your phone number and pretend to be you, so make sure to use something like Signal that has registration lock. Use services where you sign up with a username and password.
  • Don't use social media or email on the phone, and don't log into accounts unless they require the phone app as a companion (i.e. Signal). Don't put your password manager on the phone either.
  • Turn the phone off and leave it in another room away from your secure computer when it's not being used.
  • Do not use biometric authentication (fingerprint, face unlock) because they're "passwords" that are easy to steal and can't be changed.

Written out, these guidelines are simple enough, but likely requires a massive change in your behavior. However, examine the list and think about how close it is to your life pre-smartphone. Probably pretty close, right? It all boils down to "just use the phone as a communication tool, not a computer".

Cloud Services

Cloud services like Apple iCloud, Microsoft's OneDrive, and Dropbox should not be used the way they are intended. Normally, the user uploads visible, unencrypted "cleartext" data to the cloud, either to use on other devices or to have a backup. This enables the service provider to view the data put on their service, and this is not secure. For the security-minded individual, data should be fully encrypted with a key controlled by you, and only then should the encrypted data be sent to cloud services.

Obviously, the automatic backup and content upload (iCloud/Google Photos, iCloud/Google Drive, etc.) pushed on users by mobile operating systems should be disabled as well. Every "cloud" option should be off.

Services like Tarsnap do this automatically with an open-source client program, thereby proving themselves trustworthy.

Good tools to encrypt data, as mentioned in the article linked above, are 7-zip, VeraCrypt, and Microsoft's BitLocker. If you are using Windows 10 Home, you should upgrade to Windows 10 Pro to use encryption features. Microsoft does not allow Home users to be secure.

If data just needs to be synchronized between a few endpoints, Syncthing is an end-to-end synchronization program that only sends data between your devices without involving external servers.

Other Smart Devices (IoT)

Remember, the "S" in "IoT" stands for Security.

Internet of Things gadgets are generally a bad idea. There are small exceptions - mainly, the IoT gadget must be capable of operating without an Internet connection, offer a setting to enable this mode, respect that setting, and not make an attempt to send any data to the Internet. No known IoT gadget in existence works this way, so do not buy IoT gadgets unless you are a network administrator that can properly restrict them on a separate network zone.

Generally, if it is not a computer that you cannot secure, you should not trust it on your network or in your home. This includes networked appliances, like microwaves, thermostats, and refrigerators; "assistant" type devices, like Amazon's Echo or Apple's Homepod; and anything else that requires a smartphone app, a Bluetooth connection, or a Wi-Fi connection to function properly.

The more "appliance" computers you have on your network, the greater chance one of them has a vulnerability that enables bad actors to invade and compromise the network.

Compartmentalization is king, though - if you must use IoT gadgets for whatever reason, have them on a guest Wi-Fi network, and use a secondary smartphone as the IoT App device. Do not use it for personal usage.

Etc

Avoid VPNs, because you likely don't need it. It is not a good security countermeasure. Privacy, it likely does help a little bit, but not enough to matter for most people.

Avoid any "Windows 10 De-bloating" or "Disable Windows 10 Updates" or "Disable Microsoft Telemetry" guides. Most make use of third-party proprietary tools that break the security of Windows in exchange for nothing meaningful. If you don't like Windows spying on you, don't use Windows.

This goes without saying - do your updates, and don't delay them. Pick a point every week to manually do updates. This way you'll never be surprised.

Keep backups on encrypted media.

Avoid registry cleaners, program uninstallers, other utilities.

Antivirus is mostly bad, so just use the built-in Windows Defender. If you need something more robust, Malwarebytes is good because it respects the settings you set.

Paranoia

Is this not enough?

Instead of Windows, use an encrypted install of Debian, Fedora, Qubes, or OpenBSD as your operating system. Better yet, do not install an operating system on your internal disk, and use Tails on an encrypted USB thumb drive that you keep with you at all times.

Only use Open Source/Free Software whose source code you have inspected. Only use secure, encrypted, third-party-audited protocols.

Only keep sensitive data on a completely offline PC. Only transfer data to and from the system by burning a CD or DVD. The security enhancements done by minifree (sadly now defunct) provide a great set of guidelines for more adventurous users. USB as a protocol is too dangerous - fill those USB ports with hot glue or de-solder them from the motherboard.

The End...

Hopefully this guide provides a good set of tools to assist you in keeping your digital life little more secure. You don't have to implement every single step in this guide to achieve decent security, but what you should now have is a setup that was better than before. Once you've learned to live without the convenience that you get from an insecure setup, come back here, and try to add another later to your armor. Seek out other resources as well, since I don't have the answers to everything.

So, while I may have nothing left to say here, this is just the beginning for you.
