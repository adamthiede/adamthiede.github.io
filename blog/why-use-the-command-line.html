Why Use The Command Line?

[link—standalone]

When modern, flexible, decent graphical interfaces exist, many people think there's little reason to use the command line; that it's difficult to use or obsolete. This couldn't be further from the truth. The command line is powerful, verbose, easy to understand, filled with help documents and tutorials, and conceptually incredibly simple. You ask, it answers. But the main reason to use it is because, most of the time, it's faster and easier than using a graphical interface.

The command line is not always the easiest way to do things, especially for novices. But it is the fastest way to do most things, and the only way to do some things.

Using the command line for touching up a photo isn't a great idea. If you have to print or change one document, sure, use the tools where you feel at home. But if you need to resize 100 photos, or print 100 documents, or make a small change to 100 text files? You bet you're gonna need the command line. That's robot work. Are you really gonna open and close 100 pictures in your image editor, or sift through your filesystem, looking for duplicate files to get back some space? The command line will rip through these tasks in a single second where a human would take hours to do it in a graphical interface. Make the computer automate it for you!
