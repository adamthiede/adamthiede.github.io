Revisiting

[link—standalone] A blog post I saw today reminded me that the website it's on exists. This website happens to be a website I heavily referenced when first replacing the BIOS on my Thinkpad X220. It was a little over five years ago.

Much has changed in my life since then. I still have the X220, though it's been replaced (by another Coreboot-powered machine). I still use most of the software I started using around that time (except for X/i3 -> Wayland/Sway), and my usage of stable, predictable console tools has only increased. It started with the X220. I was a Linux user for a long while before I went all in on CLI-only and tiling window managers, and I'm far happier using an environment that's "mine". The meme-worthy "Thinkpad + [core/libre]boot + Arch Linux + tiling window manager" combination is actually quite comfy and Tyler's blog set me on the path.

The whole "I installed Arch Linux and now I really understand how my system works" trope is only kind of untrue. Understanding commands and components that go into building a usable desktop really helped me get a grasp of what I was working with. I had to know about, install, and configure video drivers, console configuration, bootloaders, the initramfs, disk partitioning, text editors, networking, sudo, systemd, audio, and a GUI, to name a few. Configuring these got me much more comfortable with command line editors and tmux. Rescuing a system from a live CD and doing the whole bind mounts of /proc and /sys wasn't useless - it helped me grow. Editing config files to modify my desktop was intially scary, but I am now very comfortable looking at config files, finding documentation, using man, and being able to troublshoot systems. This has not only helped me customize my desktop, but also really influenced the direction of my career and helped me land some sweet jobs.

To think it all started because I saw a thinpad and a cool window manager setup somewhere on the Internet, and thought to myself, "that's really cool! I want that!" I now use software that's stable, predictable, scriptable, and heavily configurable. Most of what I do on my computer is on the command line, in a terminal, and that journey because of some cool screenshots. I'm quite glad I fell for this meme.

Cool desktops don't change, they adapt and shift only when the user needs them to. And that's what makes them cool.
Thanks Tyler!

by Adam Thiede
