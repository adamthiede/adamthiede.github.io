It's been about a year since I first got a Pinephone. Earlier this year I attempted to use it as my main smartphone and gave up after a week. Here's my progress report for October 2021.

Too Many OS Choices

There are a lot of choices for which operating system to use on the Pinephone. A non-exhaustive list can be found here. Instead of consolidating efforts, the makers have decided to build a bunch of different OSs and let users choose how they want their phone to be broken. Most of them are the same, with minor differences. If they were to be combined, it's possible that a franken-OS of the PMOS base, with the Manjaro tweaks, would be perfect. But that doesn't exist. I'd recommend choosing either one of those, for reason explained below.

I use Phosh since it's the best performing and usable UI on the Pinephone. Plasma Mobile is very slow, confusing, and crashy. SXMO is cool, but, it's DWM for the phone, so it takes a special kind to use it. I use a tiling window manager on the desktop, but a phone is different. I've tested the following operating systems in the last month:

  • Mobian - same state as last year, basically. Things are stable, solid, and mostly work as you'd expect. Nice, but almost no progress. Stasis. It still works, but everything else works better now because of the constant progress being made on this device, and its interface.
  • PostmarketOS - the most complete operating system. Things like alarm clock, not-fully-suspending-while-playing-media, and other various auto configs work out of the box. If you're just using the device in a stock state this is the best choice. This is where most of the innovation comes from that leaks out into other distros.
  • Arch Linux - like a more up to date Mobian. Some basic OS things don't work but it's fast, up to date, and rather stable. Very upstream-y, not an opinionated configuration.
  • Manjaro - I expected not to like it, but it is my favorite currently. It is the only one in this list that lacks full disk encryption support, which is kind of table stakes for a modern mobile OS, but it manages to be blazing fast, have a decent out of the box setup, and unlike x86 Manjaro, be an actually pleasant and useful set of additions on to Arch. The UI and look and feel of just using Phosh on Manjaro is noticeably fast. It's really slick.
So yeah, I'm using Manjaro on the eMMC, set up with all my regular phone stuff, but have PMOS on my SD card since it's also fun to play with.

Cell Phone

This thing apparently makes phone calls. The same things that worked a year ago still seem to work fine - phone calls and data are a zero-configuration deal. MMS is still spotty apparently, but I do not use it or SMS. This time, my SIM card was only in the Pinephone briefly to test features, so I can't speak to anything long term.

Messaging

Telegram works great, but Telegram is not for me. Lots of xmpp clients too, since this is the Linux Desktop. There is an abundance of Matrix clients. Fluffychat is quite nice, but very basic, and the interface elements are cartoonlishly large. Fractal is laggy but usable. Nheko is barely usable. Signal, the program I've spent 5 years begging everyone in my life to use, is conspicuously abset. There are a lot of efforts trying to get it working, but nothing is sufficient. Best effort is running it in Waydroid, but that's kind of awful.

Modern messaging apps that include calling, messaging, and video chat, like Signal, WhatsApp, Viber, FB Messenger, etc. - that's not what this device is good at. Clients for these services do not exist for the Pinephone and likely never will.

Misc

Email, calendar/contact syncing, all work marvelously with Evolution's data sync daemons. Many programs exist to do various other phone things, like password management, note taking, and all that. Apps lists can be found here, at linmobapps, or here, at the mobian wiki.

Android emulators like Waydroid or Anbox have been moderately useful for bridging the gap, for things like Signal that don't have native Linux apps. But it's hideously slow on the Pinephone. Network sometimes drops in the Android container. Like the Pinephone, something for testing, but don't depend on it.

Conclusions

The Pinephone is a great PDA. It's like a Palm Pilot - it runs a limited set of discrete applications, but isn't ready for me to use as my phone. If you have already designed most of your life, and your communication (that's key) around open standards like Matrix, xmpp, and email, then it can be a usable smartphone. Otherwise, stick with an Android ROM like Calyx, Graphene, or Lineage OS without Google Apps. Almost as FOSS, and more support for what we use phones for these days.

Someday it'll be perfect. Not today, though. I'm still gonna buy the keyboard when it comes out. That might push me over the edge. Here's to hoping.
