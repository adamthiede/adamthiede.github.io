Pinephone February 2022

[link—standalone]

How's the Pinephone thing going? Glad you asked, nobody!

I write this from convergence land. Recent updates have transformed this thing from "okay-ish" to "surprisingly usable"! My phone-computer is plugged into my monitor, and I am typing with external perhipherals. It's pretty sweet. There has been an enormous amount of work by many others (and a tiny amount by yours truly!) to get this thing to a workable state. I imagine the Librem 5 or Pinephone Pro would be even better, but this is what I've got. It's still incredible.

My main SIM card lives in the Pinephone most of the time. I occasionally swap back to my Android device, but try to avoid using it otherwise. It is faster, has a working camera, calls sound clear due to HD Voice, and it can run Signal natively, but I don't like it nearly as much. The Pinephone is what I want to use.

On a daily basis, the phone does very few things when undocked - it needs to do calls, messages, music, podcasts, and maybe some email? That's about it. When docked, I treat it like a full computer. The details of my personal software stack are as follows:

  • It runs PostmarketOS Edge with the Phosh UI. I've found this to be the most "feature complete" OS that requires the least amount of hackery on my part. I used to run Arch and that worked fine too, just a little less usable, and I had to compile a lot of packages and features that PostmarketOS just has natively.
  • Chatty is for XMPP (with omemo) and SMS/MMS. It works well!
  • Dino for XMPP as a secondary client.
  • Geary and Evolution for email.
  • NhekoReborn (Flatpak) for Matrix.
  • KeepassXC with scale-to-fit for password management. It is far better than gnome-passwordsafe/"Secrets" due to the insanely high resource usage of the latter.
  • GNOME Podcasts to listen to podcasts.
  • Lollypop is for music.
  • Signal-Desktop flatpak (built from these excellent arm64 signal builds).

The battery life isn't great. I await my keyboard case anxiously. It can't really do the whole "portable hotspot" thing, but that can be easy enough to figure out.

I'm satisfied with this for now. It's not great but it is incredibly cool and useful having a full computer in my pocket. It's what I dreamed smartphones could be, but never were.

Breaking Changes

[link—standalone]

Breaking Changes are always interesting to deal with. Recently I've been trying to get a few bugs fixed in software I used, and build the resulting software. I use GNOME Podcasts on my Pinephone, and it used to have a pretty annoying bug where it would download the album art every time you viewed the media widget in the notification panel or lock screen. This was the byproduct of several bugs, both in phosh, and in gnome-podcasts. The bugs have been fixed, but it's taking a while for them to move down the pipe from git master, to release, to distributions. Sometimes, you just have to take matters into your own hands.

So a-building we will go. I really should get another Raspberry Pi to use as an arm64 build server, but until then I'll continue using the free ARM server on Oracle Cloud's Free Tier. I am not a fan of Oracle the company but they do offer a pretty nice set of resources for free, which includes ARM resources. (4 cores, 24 GB RAM, combine-able or divisible into up to 4 individual systems.) I've got the ARM resources set up just as one server with all the resources running Ubuntu.

In order to get an interactive Arch setup, I run Docker on the system, managed with Distrobox. I use the menci/archlinuxarm base image, and just exec into that. Distrobox automatically shares the home directory transparently with the container, so all the files I need are accesible both from the host system and the Arch Linux ARM container. The setup isn't automated or anything. I just write PKGBUILDs and run makepkg. Pretty primitive, but most of the software I am building has a pretty hacky, beta, manual setup anyway.

Anyway, all this to say, Meson broke on everyone recently! (issue 1, issue 2) GNOME Podcasts, like pretty much all moder GNOME/Linux Desktop software, uses Meson. It's a system I've been meaning to learn for a while but have basically zero understanding of. Browsing the issues, I figured out that I needed to imitate some of the other projects, and remove some declarations Meson's Makefile equivalent in order to get GNOME Podcasts to build:

sed -e "s/i18n.merge_file (.*$/i18n.merge_file (/" -i podcasts-$pkgver/podcasts-gtk/resources/meson.build

And now I'm a happy GNOME Podcast listener. I learned a few new things from this experience. Upstream changes happen super fast; Arch Linux doesn't always have the latest stuff (glibc on Fedora is newer!); and when you're living on the bleeding edge of software releases, waiting on critical issues and hardware support, releases seem to come really slowly!

FOSS is great because I could look at public code and public issues, figure out how to build my Podcast app myself, fix the build, install it, and use it. Thanks to all the GNOME and Meson people who figured this out before I did. I can't get over how cool it is to have a smartphone I can do this with.

Better Might Not Be Ready

[link—standalone]

or, you might not be ready for it

Recently a high-profile Internet Elder called the concept of decentralized financial systems a bad idea, mainly because of energy consumption, and because people who got into it early had a lot of money, and those who didn't care about it had none. You might not understand what I'm talking about - that's okay. This instance isn't important, but what is important is that he misses the point by a mile and a half, and is using his weight to publicly discourage improvement of a potentially better system because it isn't ready yet.

Bitcoin and Monero are good, sound, stable cryptocurrencies that make the world better. Cryptocurrency does consume a lot of energy, sure, but there are a lot of things that do, and good ideas should hardly be the first on the chopping block. The first should be the yearly smartphone and chip cycle - release a new one every 3 or 4 years, and instead of throwing the old stuff away, unlock it and open source the code, so instead of throwing batteries and metals in landfills, we can throw away planned obsolecense! If we're dead-set on decreasing the energy consumption of financial systems, why turn off the good one? How about we turn off MasterCard or Visa instead? Maybe the stock market, and betting on the fate of companies? That churns through electricity, and lots of people get hurt by it, and by the endless purusit of year-over-year growth.

The point is this:
Bitcoin and Monero are better than whatever currency your government's using. They belong to no entity but the people that use them. You can't make more. You can't cheat or steal or lie about what's going on. These systems put the power back into the hands of the people, and take it away from adversarial and oppressive organizations that want to control and change the system on their whims. The Linux Desktop is better than Windows and macOS. It supports mutiple architectures, brings decade-old computers back to business, doesn't exfiltrate your data to the NSA, and generally give the user more choice and customization than the other proprietary systems. It too gives the power to the user, making them responsible, but free.

But they aren't quite ready yet.

Photographers need Photoshop, architects need AutoCAD and SolidWorks, organizations need Active Directory support, and most people just buy a computer in a retail store and use what it comes with. There are still plenty of problems with using a Linux Desktop, though I've been exclusively using it for almost five years, and doing everything but gaming there for about ten. Although it's ready for most people, there are still plenty for whom it's not accessible - either beause of actual accessibility concerns, or just lack of knowledge.

Cryptocurrencies are the same. Just like you don't need 10,000 different choices of Linux Distribution (Hannah Montana Linux, RebeccaBlackOS, Biebian Linux, and not-Linux PonyOS are all real computer operating systems you can use!) you don't need DogeCoin or other stupid meme cryptocurrencies. Not every United State needs to have their own coin, and neither do messaging apps. At some point, we all settled on a standard for communication, and that's email. Eventually one or a few cryptocurrencies will "win" and they'll be the ones people take seriously - seems like Bitcoin, Ethereum, and Monero are all the ones that the DeFi and Privacy people can agree on, so maybe them?

Anyway, point is, maybe the things that are better don't quite hit the mark for being as functionally complete as we want yet. But that doesn't mean that the idea sucks or it's a monstrosity that's destroying lives. The idea is better, but the implementation just needs to be ironed out. Bitcoin consumes a lot of power, but Monero's algorithm is ASIC resistant and can be mined on normal CPUs, making it something that's far more attainable for regular folks. Linux is truly Free (as in freedom), but in order to live in the better brighter freer future, we need more better programs to work on Linux and other FOSS operating systems.

We have a glimpse of the future with us today, in the form of FOSS and Decentralized tech, but we still need to help it along. Get comfortable with it, use it a bunch, figure out what's wrong, and fix it! Giving up and throwing in the towel is wrong. We won't get a better world like that. Sometimes it sucks to be an early adopter, but you don't have to go all in - experiment, play around, report bugs, and get more people thinking about how to fix the problems. It's slow going but Rome wasn't built in a day. No need to use something in production that doesn't suit your needs, but don't denigrate it as a failure when it's clearly built to last. The future's just getting started.
