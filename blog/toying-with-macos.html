A mini-review of macOS from the perspective of a Linux user


A few years ago when I switched to using Linux on the desktop full-time, it was definitely not as complete and usable a desktop computing experience as the Mac. Windows 10 was easily worse (in the early days; it's become a little better, if you use the LTSC version) and the Mac was fantastic. I started using Linux more seriously after macOS 10.10 (the one with the iOS 7-like blurry flat redesign came out) and switched to it full time a couple years later. 10.9 was peak macOS for me, and it just got harder for me to stomach after that. Maximizing got taken away; I discovered tiling window managers and loved them, and even Windows has that; more settings got jumbled for no reason; iCloud became practically manadtory or forced in half the apps, and I didn't want to use it; and obviously I got super silly paranoid.
I've always kept a few spare laptops around (when I was using macOS full time, I kept a spare Windows machine for old games), and since 2017 the Mac has been in the pile of spares. I pulled it out since I'm going to be using a Mac for work soon, and wanted to see how it works nowadays.
I would give kudos to Apple for supporting recent macOS versions on an almost 8-year-old machine, but I regularly use the latest versions of Fedora and Ubuntu on machines that are older than that, and they perform better, so that's moot.
It does perform surprisingly well on a standard 750GB HDD, and save for long boot times, operates as smoothly as it does on an SSD. It's very cool and quiet, has decent battery life, charges fast, doesn't stutter and operates well doing daily tasks. All the hardware works and it has a plethora of ports, a little LED battery meter on the case, and a DVD drive in a very compact package. It's not really all that impressive, since that's the pretty low bar I expect things to clear, but it gets the basics right.
The GUI is fine. I really do like tiling and workspaces, and the Mac is okay at those. I have to use the trackpad instead of the keyboard, but I assume there are decent keyboard shortcuts I've not looked up for things like "go to workspace n" or "move application to workspace n". I hear it has picture-in-picture, but have not tried it yet. I wonder if I could do that with a terminal instead of a video. Overall, it's less configurable than GNOME3, even, and that's saying something. Every little app costs money (things that are just normal GNOME extensions or built-in features) and I don't want to pay for something that might not be good, or might be spyware.
The worst part is the lack of package management. I rely HEAVILY on dnf/apt/flatpak/snap to do automatic updates for all my programs/system at once, and lacking a good package manager built-in is a huge flaw of the platform. I've installed Homebrew since it's what I remember using prior, and it's still good. It'll take some getting used to. Nice to have an open-source way to install and update things like keepassxc, wireshark, ffmpeg, tmux, etc.
The best part is that it's a Unix system (I know this!) and the terminal application is REALLY GOOD. They've switched to ZSH by default (seriously? Why?) but it's easy to switch back to bash or fish. I could probably install a plethora of ncurses apps on a VPS and live in an SSH connection and be perfectly happy with macOS.
Conclusion: I'm a Free Software diehard, so I'll keep using Ubuntu/Debian/Fedora forever, but macOS is good if you don't have the option to use Desktop Linux. Since I don't have to pay for the inevitable keyboard repair (it being a work computer), I guess I'll be content with macOS!
by Adam Thiede
