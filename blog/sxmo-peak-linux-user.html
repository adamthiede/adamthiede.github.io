SXMO - Peak Linux User

[link—standalone]

It was only a matter of time.

Back in February, I gave sxmo a serious try for the first time. I didn't quite get it and it didn't really "click" for me. I'd tried it before, because I've tried every postmarketOS interface at some point, but this was the first time I really truly gave it a go.

You know where this story ends. Like everything good in *NIX-land, it has a high initial learning curve, but is more lightweight, customizable, and ultimately, more user-friendly by virtue of being hackable. I use sway on the desktop, and now I'm using it on my phone, too. I don't have to understand how to use two different interfaces, becasue they're the same! It's like the people that use an iPhone and an iPad-as-laptop - it's the same interface between multiple devices.

Tiling window managers can actually make a lot of sense on smaller screens:

  • window decorations usually end up being smaller or nonexistent
  • it's easy to add in customizable launchers
  • scripting is a "first-class citizen", leading to less actions per task (even Apple thinks it's important)
  • window managers are lighter weight than a full "desktop" environment, leading to longer battery life
(This is only kind of a joke.)

A tiling window manager on a phone is peak Linux ricer. It's unironically hilarious. But also, seriously, this is the most fun I've had in a while. Being able to ssh into my phone and control its functions from an ncurses-like interface is incredible. Using the buttons in a different way, contextually, is also really cool. Before sxmo, the keyboard case was only kind of okay, but with sxmo, the keyboard case actually makes a lot of sense. This is a really natural way to use the pinephone - why else would one have this thing but not for hacking around in a totally weird Linux environment?

There are, of course, a few changes I've had to make to the environment, shortcuts, hooks, libraries and programs I've compiled from source and stuffed into my ~/.config directory... but that's what this was designed for. It's why I bought a Linux Phone. I'm really happy to be able to treat my telephone like a computer, instead of an appliance. Every aspect is a hacker's dream come true.

by Adam Thiede
