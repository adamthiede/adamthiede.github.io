Raspberry Pis and Spades

[link—standalone]

Raspberry Pis are great little computers. They work well, don't break the bank, and there's a lot of accessories and tutorials for beginners out there to get started. They can be a desktop, server, or portable system. There's no end to what you can do with them. People have used them for all sorts of things over the years.

Recently the Raspberry Pi Foundation automatically, silently, added a Microsoft repo to the operating system the provide, Raspberry Pi OS. They're proud of the fact that VSCode is now available for the Pis, but others were not so happy about it, claiming it was a "secret" addition and it enables Microsoft telemetry. As always, the truth is somewhere in the middle, and it's kind of boring. Yes, they added a repo to Raspberry Pi OS. Yes, you can disable it pretty easily. No, Microsoft's repo isn't going to replace your network drivers with spyware (although the capability is now there). They should not have added it on "Lite" (no desktop, console-only) installations of the OS or on hardware like the Pi Zero, because they wouldn't be able to run it anyway.

But let's call a spade a spade: the Raspberry Pi's website is all bout how you can use these computers to teach kids to code. They offer simple tools and tutorials to help get complete beginners started using computers. They're supposed to be easy coding/computer on-ramps for educational purposes. They're not meant to be cheap servers for experienced Linux users, which is how I see them most often used in my circles!

There should be no anger here. Just a realization that we, the admins, the people who noticed the new repo, were never the target market for these devices. We've been skating by for a while using these things as cheap "lite" home servers, running pi-hole, VPNs, and media servers, many of us not realizing we were shoving a round peg into a square hole.

The short-term solution for people like me is clear. If you must use a Raspberry Pi, at least don't use Raspberry Pi OS - it's almost Debian, but not quite. Debian does make official images for the Raspberry Pis, and do does Ubuntu. Pick your poison. They both work great. Their policies are clear on what software they include. The Raspberry Pi images are supported in the same way their x86 OS is. Using Raspberry Pi OS and pretending it's Debian is clearly not going to work for much longer, for the people that just want Debian.

Long-term, if what we want is a server, let's use server hardware and server OSs. We should be buying NUCs, PCEngines boards, Pine64 Single-Board Computers, or building our own home servers - things that were created with "power users" in mind. Not toys built for kids. Personally I've transitioned running all my Pi's network-critical systems into containers or VMs on my NAS, and the Pi is now a toy, or a game server. I'm using it for fun, to play, and to learn. (It also now runs Ubuntu, because I can expect consistency there, not unexpected changes I don't appreciate just to make it easier for kids to use.) Raspberry Pis are great, but let's call a spade a spade. They're not for us, and never were. We shouldn't expect them to be.

by Adam Thiede
